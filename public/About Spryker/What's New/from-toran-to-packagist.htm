Using Packagist For Module Download

Previously we used Toran Proxy in our code hub code.spryker.com. Since Toran Proxy has been deprecated and is being phased out, we have started to use public Packagist to host our modules.

From now on, please download all Spryker modules from Packagist.

The new tool is automatically already available without any change in your composer.json thanks to Packagist being the default lookup for Composer.

However, the deprecated code.spryker.com URLs should be removed from your composer.lock to avoid the deployment process being slowed down by lookup misses once Toran becomes unavailable.

First of all, remove the code.spryker.com block from your composer.json repositories section:


"repositories": [
	{
		"type": "composer",
		"url": "https://code.spryker.com/repo/private"
	}
],

Then run composer update --lock to generate a fresh lock file with updated download URLs. The important part here is that --lock prevents any update of modules. So you are not accidentally upgrading any module here. That's it. Enjoy the new power of Packagist.

To further speed up the composer update process, use this composer tool. After running composer global require hirak/prestissimo, the next composer update or composer install command should be way faster for both local development and server deployment.
