Product TTL (Time To Live)

With TTL you can easily define from and till when your product(s) will be visible and available for purchase by your customers, without having to manually manage your inventory. This feature is extremely useful if you sell products with expiry date, seasonal products or products that are valid for a certain period of time (e.g. tickets). If for promotional reasons, you temporally sell some products within a limited time frame, or if you want to start selling some products (e.g. a collection) from a specific date and time. The product validity time can be set to up-to-the minute accuracy.

TTL For Products Overview

The TTL feature is implemented through isActive field by introducing two more fields for product concretes in the Administration Interface - "Valid from" and "Valid to". Product can have exactly one "life" - one (or no) start, one (or no) end. If "Valid from" date has not come yet, or if "Valid to" has been reached, the value of isActive field is set to "false" and the product is invisible for customers. This being said, the product validity settings overrule manual (de)activation of products, which means that, if, for example, a product has been manually set to "Deactivated" but has "Valid from" and Valid to" values specified, and "Valid to" lies in the future, the product will be displayed until after the time set in "Valid to" field. To deactivate the product in this case, it would be necessary to clear the values in "Valid from" and "Valid to" fields.

Time set for the product validity "Valid from" and Valid to" fields is accurate up to the minute, however technically it is possible to change the accuracy (up to the second or up to the days) via a cronjob. The time is set in GMT by default.

Use Cases

The table below shows possible usage scenarios with the TTL feature and resulting system behavior.

Validity dates settings Result
"Valid from" is set, "Valid to" is set and lies in the future Product goes online starting from "Valid to" time and remains online until after the "Valid to" time
"Valid from" is set, "Valid to" is not set Product goes online from "Valid from" time and never goes offline
"Valid from" is not set, "Valid to" is set Product goes online after manual activation and remains online till "Valid to" time is reached
"Valid to" is set in the past Product will be deactivated shortly
"Valid from" is set, "Valid to" is set and lies in the future, the product is deactivated manually Message saying that the product will be activated because of set "Valid from" and "Valid to" time
