ZedRequest

ZedRequest facilitates the connection between the frontend application (Yves) to the backend (Zed). The connection is a HTTP POST request with a transfer object as a payload. It provides a client that is used by almost all the other clients in order to send requests from the frontend application and get responses from the backend.

 

Latest version: 3.5.1 View on GitHub
Last update: 15 Nov 2018
 
  Type: Core Splitted
Group: Infrastructure

API (Facades)

Facade Method Description
getRepeatData

  • Method intended to be used in development environment only!

  • Returns the request data from last yves to zed request

  • If mvc given then it returns the data matching to the mvc argument

  • mvc must be like bundle_controller_action

Clients

Client Method Description
call

  • Prepare and make the call to Zed.

    Third argument has changed from int to array. BC compatibility method will

    convert the previous accepted integer to ['timeout => $timeoutInSeconds]
getLastResponseInfoMessages

  • Returns an array of MessageTransfers containing info messages for the last response.
getLastResponseErrorMessages

  • Returns an array of MessageTransfers containing error messages for the last response.
getLastResponseSuccessMessages

  • Returns an array of MessageTransfers containing success messages for the last response.
addFlashMessagesFromLastZedRequest

  • Get messages from Zed request and put them to session in next order:

  • Writes error message to flash bag.

  • Writes success message to flash bag.

  • Writes informational message to flash bag.

Dependencies:

  • PHP >=7.1
  • Application ^3.0.0
  • Config ^3.0.0
  • Guzzle ^2.1.0
  • Kernel ^3.0.0
  • Log ^3.0.0
  • Messenger ^3.0.0
  • Silex ^2.0.0
  • Symfony ^3.0.0
  • UtilNetwork ^1.1.0
  • UtilText ^1.1.0
