CmsBlockCategoryConnector

CmsBlockCategoryConnector extends the CmsBlock module to allow category block management. Category blocks are blocks that can be embedded into the category template. It is possible to specify on which specific categories we want them to be rendered.

 

Latest version: 2.1.4 View on GitHub
Last update: 05 Oct 2018
 
  Type: Core Splitted
Group: Supportive

API (Facades)

Facade Method Description
updateCmsBlockCategoryRelations

  • Perform actions based on CMS Block transfer

  • Delete all relations categories to cms blocks

  • Add new relations defined in the transfer object
updateCategoryCmsBlockRelations

  • Perform actions based on Category transfer

  • Delete all relations categories to cms blocks

  • Add new relations defined in the transfer object
hydrateCmsBlockCategoryRelations

  • Hydrate Cms Block with an array of related category IDs
getRenderedCategoryList

  • Generate rendered list of assigned categories
getCmsBlockCollection

  • Get collection of related CMS Blocks
syncCmsBlockCategoryPosition

  • Takes positions from configuration

  • Creates new CMS Block category position records

  • Does not remove/update existing records
findCmsBlockCategoryPositionByName

  • Finds a position by name

  • Hydrates transfer object

  • Returns NULL if position does not exist
runStorageCmsBlockCategoryCollector

  • Hydrate CMS Block to Category relation with block names

  • Collect relation to Storage

Dependencies:

  • PHP >=7.1
  • Category ^4.0.0
  • CmsBlock ^1.2.0 || ^2.0.0
  • Collector ^5.0.0 || ^6.0.0
  • Gui ^3.6.0
  • Kernel ^3.0.0
  • Locale ^3.0.0
  • PropelOrm ^1.0.0
  • Symfony ^3.0.0
  • Touch ^3.0.0 || ^4.0.0
  • UtilDataReader ^1.0.0
  • UtilEncoding ^2.0.0

Suggested Installations:

  • CmsBlockGui ^1.0.0 || ^2.0.0
