CmsBlock

CmsBlock supports management of blocks. Blocks can be viewed as fully localizable partial content snippets that can be embedded in other pages such as Product pages, Category pages, and so on. The CMS Block module supports flexible extension by adding a new connector.

 

Latest version: 2.1.1 View on GitHub
Last update: 26 Oct 2018
 
  Type: Core Splitted
Group: Functional

API (Facades)

Facade Method Description
findCmsBlockById

  • Returns an CMS Block transfer by id
activateById

  • Changes status of the block to Active
deactivateById

  • Changes status of the block to Inactive
updateCmsBlock

  • Updates CMS Block

  • Asserts that chosen template still exists
createCmsBlock

  • Creates a CMS Block record

  • Asserts that chosen template still exists

  • Sets CMS Block ID to transfer
syncTemplate

  • Loops by all template files in passed path

  • Creates records with templates in DB
findGlossary

  • Find CMS Block glossary

  • Hydrate glossary placeholders

  • Hydrate placeholder translations
saveGlossary

  • Creates new or updates nested placeholders
createTemplate

  • Validate path on existing template file

  • Create a new template record
findTemplate

  • Returns template transfer if it exists

  • Return NULL if no record in DB for template with this path
hasTemplateFileById

  • Finds a template in DB

  • Checks template file in template directories

Clients

Client Method Description
findBlocksByNames

  • Find blocks by provided array of names with a single multi request to a storage
findBlockNamesByOptions

  • Find blocs by required options

  • Return only block names which fit to all provided options
generateBlockNameKey

  • Prepare a valid block key by provided name
